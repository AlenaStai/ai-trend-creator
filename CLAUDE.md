# AI Trend Creator — архитектура проекта

## Что это

Личный инструмент для автора AI-блога (Telegram-канал про нейросети: их использование в жизни,
в работе и заработок на них). Решает конкретную проблему: не успевать следить за трендами
в TikTok/Instagram/YouTube и при этом регулярно выпускать контент про нейросети, который
резонирует с актуальной повесткой — и сразу получать под это готовое короткое видео.

Цепочка: **Trends → Analyze (с оценкой виральности) → Generate Content (виральный сценарий) → Generate Video (HeyGen Video Agent) → Экспорт**

Публикация в соцсети сознательно вне цепочки — на выходе готовый файл видео + текстовый
пакет (сценарий, caption), дальше автор публикует сама, где и как захочет.

Это выпускной проект курса, срок разработки — 10-11 дней. Ниже чётко разделено,
что входит в MVP, а что — нет. Не выходи за границы MVP без явного запроса.

## Границы MVP (важно!)

**Входит:**
- Сбор трендов через Trends MCP (Google Trends, YouTube, TikTok, Reddit — одно подключение,
  без скрапинга) как основной источник + Reddit API/YouTube Data API/RSS как фолбэк
- Анализ и скоринг трендов через Claude API: вероятность вирусности с явной меткой
  "вероятно залетит / под вопросом / вряд ли залетит" и обоснованием, релевантность нише,
  конкуренция
- Генерация виральных сценариев: хук в первые 2 секунды, конкретика вместо общих слов,
  структура с петлёй/неожиданным поворотом — под личный авторский стиль
  (см. раздел "Стиль контента"). Этот сценарий становится промптом для видео-шага
- Генерация ОДНОГО демо-видео целиком через HeyGen Video Agent: сценарий → визуал
  (motion graphics/сток/AI-картинки) → аватар → монтаж → готовый ролик, один инструмент,
  без отдельной сборки
- Экспорт готового пакета: видеофайл + текст (сценарий, caption, image-промпты)
- Простой дашборд (Streamlit) с кнопками по шагам цепочки

**НЕ входит в MVP (сознательно вне скоупа):**
- Автопостинг в любую соцсеть, включая YouTube
- fal.ai / InVideo AI / Hyperframes/mcp-video — не нужны, Video Agent закрывает всю
  видео-часть цепочки одним инструментом
- Аналитика по результатам опубликованного контента

Если возникает соблазн расширить скоуп — сначала закончить цепочку end-to-end на одном
тренде, потом уже расширять.

## Стиль контента

Ключевое отличие инструмента от generic AI-генератора — тексты и сценарии звучат
как ты, а не как нейросеть. Промпт в `content_generator.py` должен явно требовать:

- Короткие рубленые фразы, разговорный ритм — это важно и для Video Agent тоже
  (длинные предложения хуже ложатся на темп и паузы в готовом видео)
- Самоирония, без пафоса
- Без симметричных конструкций ("не просто X, а Y")
- Без инфоблогерских CTA ("сохрани", "поставь лайк если согласен")
- Хук — первые 2 секунды решают всё: конкретный факт/цифра/неожиданность, не разгон
- Пауза после ключевых мыслей — держать это в сценарии явно

Проверять результат по принципу `/humanizer`: если текст можно было написать про любой
другой AI-канал — промпт не сработал, переписывать. Готовый сценарий передаётся в HeyGen
Video Agent как есть — чем чище и конкретнее текст на этом шаге, тем меньше правок
понадобится в видео.

## Технологический стек

- **Backend/логика**: Python 3.11+
- **UI**: Streamlit
- **База данных**: SQLite (файл, без внешней инфраструктуры)
- **AI-анализ и генерация текста**: Claude API (Anthropic)
- **Сбор трендов**: Trends MCP (trendsmcp.ai) — основной источник; Reddit API/YouTube Data
  API/RSS — фолбэк
- **Генерация видео целиком**: HeyGen MCP, Video Agent — сценарий, визуал, аватар,
  монтаж одним вызовом

## MCP-серверы

**Trends MCP** (trendsmcp.ai) — живые данные трендов из Google Trends, YouTube, TikTok,
Reddit и других источников через одно подключение, без скрапинга. Бесплатный тариф —
100 запросов/месяц, 20/день. Это сторонний агрегатор, не официальный TikTok API —
надёжность стоит проверить в первые дни разработки (день 1-2); если нестабилен —
переключаемся на фолбэк (Reddit + YouTube + RSS).

**HeyGen MCP — Video Agent** — прежняя связка "HeyGen для хука + отдельный инструмент
для b-roll и сборки" заменена на один инструмент. Video Agent принимает сценарий/промпт
и сам: пишет разбивку на сцены, подбирает визуал (motion graphics, сток-футаж или
AI-картинки), кастит аватар, синхронизирует голос, монтирует и расставляет темп —
всё в одном вызове.

Технические детали:
- **Кредиты**: Video Agent списывает 20 премиум-кредитов за минуту видео — из того же
  пула, что и обычная генерация аватара, план Creator (~$24-29/мес, ≈1900-2300 ₽)
  ничего сверху не требует
- **Формат**: явно указывается Portrait (вертикальный) при генерации — под Shorts/Reels
- **Аватар**: свой цифровой двойник создаётся один раз заранее через официальный
  HeyGen-скилл для Claude Code; Video Agent может использовать его вместо авто-подбора
- **Риск несовпадения визуала**: Video Agent сам выбирает визуал под текст — если
  результат не устроил, можно доправить прямо в чате с агентом ("смени визуал в сцене 2")
  или открыть в AI Studio на таймлайне для точной правки — это и есть страховка от
  ситуации "AI подобрал не то", которая была главной причиной отказа от InVideo AI
- **Сценарий под Video Agent**: короткие фразы, паузы — то же, что уже заложено
  в промпт content_generator.py (см. "Стиль контента")

## Практические риски и детали

**Бюджет — только HeyGen.** План Creator $24-29/мес (≈1900-2300 ₽) — единственная
платная часть, и она уже покрывает и аватар, и Video Agent из одного пула кредитов.
Claude API (анализ + генерация текста) — копейки (~$5-15 за весь цикл тестирования).
Trends MCP, Reddit API, YouTube Data API — бесплатно. HeyGen — зарубежный сервис,
российская карта может не пройти напрямую — проверить способ оплаты в день 1-2.

**Демо не должно генерироваться вживую.** Рендер у Video Agent занимает минуты —
готовить заранее один полностью сгенерированный и сохранённый пример для показа
на защите, живую генерацию показывать только если время на защите позволит.

**Git и .gitignore.** `.env` вне репозитория с первого коммита — создать `.gitignore`
в день 1, до git add.

## Переменные окружения (.env)

```
ANTHROPIC_API_KEY=
REDDIT_CLIENT_ID=
REDDIT_CLIENT_SECRET=
TRENDS_MCP_API_KEY=
```

## План разработки по дням

**День 1-2 — фундамент**
- Git init + .gitignore (с .env) — до первого коммита
- Структура проекта, requirements.txt, базовый Streamlit-каркас с 4 разделами
  (Trends / Analyze / Generate Content / Generate Video)
- Создать HeyGen-аватар через официальный скилл — одноразовая настройка; заодно
  проверить способ оплаты (зарубежная карта/сервис-посредник)
- Подключить Trends MCP, сразу проверить стабильность на реальных запросах
- Параллельно набросать фолбэк-коллекторы (Reddit, YouTube, RSS)
- Единая лента трендов в SQLite

**День 3-4 — анализ**
- Claude API для скоринга: вероятность вирусности с явной меткой (залетит/под
  вопросом/вряд ли) и обоснованием, релевантность нише, конкуренция, причина тренда
- Это ключевая фишка проекта — стоит потратить время на промпт

**День 5-6 — генерация контента и стиль**
- Виральный сценарий (хук за 2 секунды, конкретика, петля/поворот), caption,
  image-промпты
- Промпт явно требует авторский стиль (см. "Стиль контента"), проверять по принципу
  `/humanizer`

**День 7 — видео**
- Интеграция с HeyGen Video Agent: передаём сгенерированный сценарий, Portrait-формат,
  свой аватар
- Если результат не устроил — доправить через чат с агентом или AI Studio, а не
  перегенерировать с нуля (экономит кредиты)
- Сразу сохранить один удачный рендер как готовый пример для защиты — не полагаться
  на генерацию вживую на демо

**День 8 — экспорт**
- Кнопка экспорта: видеофайл + текстовый пакет (сценарий, caption, image-промпты)

**День 9-11 — запас**
- Полировка интерфейса, README, отладка багов, подготовка демо-сценария для защиты
- Прогнать весь сценарий несколько раз подряд перед защитой — исключить сюрпризы
  от HeyGen и Trends MCP
- Расширенный буфер — видео-шаг упростился до одного инструмента, освободившееся
  время не жалко потратить на полировку анализа и стиля текста

## Стиль кода

- Простой, читаемый Python, без излишней абстракции — это MVP, не продакшн-система
- Каждый collector возвращает список словарей одинаковой структуры
  (title, source, url, score_raw, published_at) — чтобы анализатор работал единообразно
- Ошибки API не должны ронять всё приложение — если один источник недоступен,
  остальные продолжают работать
